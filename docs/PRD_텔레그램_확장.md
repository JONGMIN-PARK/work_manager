## 2. 확장 기능 개요

기존 앱의 풍부한 데이터(일정, 체크리스트, 수주, 문서, 통계)를 텔레그램으로 확장하여, 웹 접속 없이도 핵심 업무를 파악하고 간단한 액션을 수행할 수 있게 한다.

### 2.1 목표

| 목표 | 측정 기준 |
|------|-----------|
| 텔레그램만으로 일상 업무 파악률 80% | 명령어 사용 빈도 / 웹 로그인 빈도 |
| 이벤트 누락 알림 제로 | 모든 핵심 이벤트에 알림 트리거 연결 |
| 양방향 인터랙션 지원 | 텔레그램에서 상태 변경 가능 |

## 3. 신규 봇 명령어

### 3.1 일정 관련

| 명령어 | 기능 | 우선순위 |
|--------|------|----------|
| `/calendar` | 이번 주 일정 (회의, 출장, 납기, 연차 등) | P1 |
| `/calendar 14` | 향후 N일 일정 | P1 |
| `/today` | 오늘의 일정 + 이슈 + 마감 통합 브리핑 | P1 |

**출력 예시: /calendar**
```
📅 이번 주 일정

3/25(화)
  🤝 09:00 SK하이닉스 주간회의
  🏁 B2024-001 납기일

3/26(수)
  ✈️ 삼성 대구공장 출장
  🛠️ 정기점검 — A라인

3/28(목)
  🌴 김철수 연차
```

**데이터 소스**: `events` 테이블 (title, type, start_date, end_date, assignees)

---

### 3.2 체크리스트 / 할 일

| 명령어 | 기능 | 우선순위 |
|--------|------|----------|
| `/tasks` | 내 프로젝트의 미완료 체크리스트 항목 | P1 |
| `/checklist <프로젝트명>` | 특정 프로젝트 체크리스트 진행 현황 | P2 |
| `/done <항목번호>` | 체크리스트 항목 완료 처리 (양방향) | P2 |

**출력 예시: /tasks**
```
📝 미완료 작업 (8건)

[B2024-001] SK하이닉스 2호기
  ⬜ 1. 카메라 보정 완료
  ⬜ 2. 통신 테스트
  ✅ 3. 기구 조립 (완료)

[B2024-003] LG에너지 검수
  ⬜ 4. 검수 체크시트 작성
  ⬜ 5. 고객 입회 일정 확인

💡 완료: /done 1
```

**데이터 소스**: `checklists` 테이블 (project_id, phase, items JSONB)

### 4.3 일일 브리핑 (event_today)

매일 오전 9시 자동 발송되는 개인 맞춤 브리핑:

```
☀️ 홍길동님, 좋은 아침입니다!

📅 오늘 일정
  🤝 09:00 SK하이닉스 주간회의
  🏁 B2024-001 납기일

🔴 긴급 이슈 1건
  · 카메라 보정 오류 (SK하이닉스)

📝 미완료 작업 3건

좋은 하루 되세요!
```

### 4.4 주간 다이제스트 (weekly_digest)

매주 월요일 오전 발송:

```
📊 3월 3주차 리포트

⏱ 팀 총 투입: 412h (8명)
📋 신규 수주: 2건
✅ 완료 이슈: 7건 | 🔴 신규: 3건
📈 프로젝트 진행: 완료 1, 지연 2

⚠️ 주의사항
  · B2024-001 납기 D-5
  · 김철수 과부하 (일평균 10.2h)
```

## 5. 인라인 버튼 (양방향 인터랙션)

### 5.1 이슈 알림 → 즉시 액션

이슈 배정 알림에 인라인 키보드 추가:

```
🔴 이슈 배정
SK하이닉스 — 카메라 보정 오류
담당: 홍길동

[대응 시작]  [상세 보기]
```

| 버튼 | 동작 |
|------|------|
| `대응 시작` | 이슈 상태 → inProgress 변경 |
| `상세 보기` | 이슈 상세 정보 전송 |

### 5.2 체크리스트 완료

```
📝 체크리스트 — SK하이닉스 2호기 (검수)

⬜ 카메라 보정 완료    [완료 ✓]
⬜ 통신 테스트          [완료 ✓]
✅ 기구 조립
```

### 5.3 일정 응답

```
📅 내일 10:00 주간회의
참석자: 홍길동, 김철수, 박영희

[참석]  [불참]  [10분 전 알림]
```

## 6. 그룹 채팅방 지원

### 6.1 팀/프로젝트 채널 연동

| 기능 | 설명 |
|------|------|
| 프로젝트 채널 | 프로젝트별 그룹 채팅방에 봇 추가 → 해당 프로젝트 알림만 수신 |
| 팀 채널 | 부서별 그룹 → 팀 관련 알림 집중 |
| 공지 채널 | 전체 공지용 (신규 수주, 납품 완료 등) |

### 6.2 DB 스키마 확장

```sql
CREATE TABLE IF NOT EXISTS telegram_group_links (
  id          SERIAL PRIMARY KEY,
  chat_id     BIGINT NOT NULL UNIQUE,
  link_type   VARCHAR(20) NOT NULL,      -- 'project' | 'team' | 'announce'
  link_id     VARCHAR(100),              -- project_id 또는 department_id
  linked_by   UUID REFERENCES users(id),
  linked_at   TIMESTAMPTZ DEFAULT NOW(),
  is_active   BOOLEAN DEFAULT TRUE
);
```

## 9. 기술 고려사항

| 항목 | 대응 |
|------|------|
| 메시지 길이 제한 (4,096자) | 긴 목록은 페이지네이션 ("다음 5건: /more") |
| 인라인 버튼 콜백 | `callback_query` 핸들러 추가, `answerCallbackQuery` 응답 |
| 그룹 채팅 봇 권한 | BotFather → `/setjoingroups` Enable, Privacy mode OFF |
| 스케줄러 안정성 | `setTimeout` → `node-cron` 패키지로 교체 권장 |
| 양방향 데이터 변경 | 낙관적 락(optimistic lock) 적용, 충돌 시 알림 |
| Rate limit | Telegram: 30 msg/sec, 그룹: 20 msg/min → 큐 기반 발송 |

## 11. 성공 지표

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 일일 브리핑 열람률 | 80% | notification_logs 발송 성공률 |
| 봇 명령어 일일 사용 | 인당 3회+ | Webhook 로그 집계 |
| 인라인 버튼 사용률 | 50%+ | callback_query 로그 |
| 웹 로그인 감소 | 20% 감소 | audit_logs 비교 |
| 알림 → 액션 전환율 | 30%+ | 알림 후 5분 내 상태 변경 비율 |
