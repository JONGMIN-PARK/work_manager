# PRD: 프로젝트 달력 & 타임라인 관리

> 업무일지 분석기 v6.0 — 추가 기능 기획서
> 작성일: 2026-03-11 | 대상: 관리자(팀장)

---

## 1. 배경 및 목적

### 현재 상황
- 업무일지 분석기는 **과거 데이터 분석**(주간/월간/트렌드)에 특화되어 있음
- 프로젝트 일정, 마일스톤, 향후 계획을 **한눈에 조망**할 수 있는 뷰가 없음
- 수주번호별 프로젝트가 언제 시작/종료되는지, 현재 어디쯤인지 파악이 어려움

### 목표
| # | 목표 | 설명 |
|---|------|------|
| G1 | **일정 가시성** | 달력 뷰에서 프로젝트 일정, 마일스톤, 팀 이벤트를 한눈에 파악 |
| G2 | **타임라인 조망** | 간트 차트 형태로 다수 프로젝트의 시작~종료 기간을 비교 |
| G3 | **업무-일정 연계** | 기존 업무일지 데이터와 프로젝트 일정을 연결하여 진척률 자동 산출 |
| G4 | **관리자 대시보드** | 팀장이 팀 전체 상황을 빠르게 판단할 수 있는 통합 뷰 |

## 3. 기능 상세

### 3.1 프로젝트 달력 (Calendar View)

#### 탭 추가
- 기존 탭: `주간 분석` | `아카이브` | `트렌드`
- **추가 탭**: `📅 달력` | `📊 타임라인`

#### 달력 UI
```
┌─────────────────────────────────────────────┐
│  ◀  2026년 3월                    주간 │ 월간 │
├──────┬──────┬──────┬──────┬──────┬──────┬──────┤
│  일  │  월  │  화  │  수  │  목  │  금  │  토  │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│      │      │      │      │      │      │   1  │
│      │      │      │      │      │      │      │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│   2  │   3  │   4  │   5  │   6  │   7  │   8  │
│      │ ██ A │ ██ A │ ██ A │      │      │      │
│      │      │ ██ B │ ██ B │ ██ B │ ██ B │      │
├──────┼──────┼──────┼──────┼──────┼──────┼──────┤
│  ... │ ...  │ ...  │ ...  │ ...  │ ...  │ ...  │
└──────┴──────┴──────┴──────┴──────┴──────┴──────┘
  ██ = 프로젝트 바 (수주번호별 색상)
```

#### 일정 등록 모달
| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 제목 | text | Y | 일정명 |
| 유형 | select | Y | `마일스톤` / `회의` / `납기` / `출장` / `기타` |
| 시작일 | date | Y | |
| 종료일 | date | N | 미입력 시 시작일과 동일 (하루 일정) |
| 프로젝트 | select(multi) | N | 수주번호 목록에서 선택 (order.js 연동) |
| 담당자 | select(multi) | N | 팀원 목록에서 선택 |
| 색상 | color | N | 자동 배정 또는 수동 선택 |
| 메모 | textarea | N | 상세 내용 |
| 반복 | select | N | `없음` / `매주` / `격주` / `매월` (P2) |

#### 뷰 모드
- **월간 뷰** (기본): 전통적 달력 그리드, 프로젝트 바가 날짜 셀에 가로로 표시
- **주간 뷰**: 7일 확대, 시간대별 슬롯 표시

#### 필터
- 프로젝트(수주번호)별 필터
- 담당자별 필터
- 일정 유형별 필터
- 기존 팀원 그룹 필터 연동

### 3.2 프로젝트 타임라인 (Gantt View)

#### 타임라인 UI
```
프로젝트              1월    2월    3월    4월    5월    6월
──────────────────────┬──────┬──────┬──────┬──────┬──────┬──────
S2024-001 ○○설비     │██████████████│      │      │      │      │
  └ 설계              │██████│      │      │      │      │      │
  └ 제작              │      │██████│      │      │      │      │
  └ 납품              │      │      │◆     │      │      │      │
──────────────────────┼──────┼──────┼──────┼──────┼──────┼──────
D2025-010 △△시스템   │      │      │████████████████████│      │
  └ 개발              │      │      │██████████████│      │      │
  └ 테스트            │      │      │      │      │██████│      │
  └ 배포              │      │      │      │      │      │◆     │
──────────────────────┼──────┼──────┼──────┼──────┼──────┼──────
                      │      │      │ ▼ 오늘│      │      │      │
                      ◆ = 마일스톤   ▼ = Today Line
```

#### 프로젝트 등록/편집
| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 수주번호 | text/select | Y | 신규 입력 또는 기존 수주번호 선택 |
| 프로젝트명 | text | Y | order.js 연동, 자동 완성 |
| 시작일 | date | Y | |
| 종료일 | date | Y | |
| 상태 | select | Y | `대기` / `진행중` / `지연` / `완료` / `보류` |
| 진척률 | number | N | 0~100%, 업무일지 연동 시 자동 산출 |
| 담당자 | select(multi) | N | 팀원 지정 |
| 색상 | color | N | 수주번호별 자동 배정 |

#### 마일스톤 (하위 단계)
| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 단계명 | text | Y | 예: 설계, 제작, 테스트, 납품 |
| 시작일 | date | Y | |
| 종료일 | date | Y | |
| 상태 | select | Y | 상위 프로젝트와 동일한 상태 옵션 |

#### 타임라인 인터랙션
- **확대/축소**: 일/주/월/분기 단위 스케일 전환
- **드래그 스크롤**: 좌우 드래그로 기간 이동
- **바 드래그 (P1)**: 프로젝트 바 드래그로 일정 변경
- **Today Line**: 빨간 수직선으로 오늘 날짜 표시
- **지연 표시**: 종료일 초과 프로젝트는 바 색상 변경 (빨간색 점선)
- **툴팁**: 바 호버 시 상세 정보 표시

#### 업무일지 연동 (진척률 자동 산출)
```
진척률 = (해당 수주번호 누적 투입시간 / 예상 총 투입시간) × 100
```
- 예상 총 투입시간은 프로젝트 등록 시 수동 입력
- 아카이브 데이터에서 수주번호 매칭하여 누적시간 자동 집계
- 진척률 바 위에 % 표시

## 5. UI/UX 설계 원칙

### 기존 시스템과의 일관성
- 기존 **테마 7종** 전부 적용 (config.js `TH` 연동)
- 기존 **색상 팔레트** (`COL`) 활용
- 기존 **팀원 별칭/그룹** 시스템 연동
- 기존 **수주번호 ↔ 수주명** (order.js) 연동
- 탭 전환 방식 유지, 모달 디자인 패턴 동일 적용

### 달력 디자인
- 프로젝트 바는 수주번호별 자동 색상 배정 (COL 팔레트 기반)
- 마일스톤은 ◆ 다이아몬드 마커로 표시
- 오늘 날짜 셀 강조 (테마별 액센트 색상)
- 지연 프로젝트는 빨간 점선 테두리로 시각적 경고

### 반응형
- 기존 앱과 동일한 반응형 브레이크포인트 유지
- 모바일에서는 달력 리스트 뷰 폴백

## 7. 구현 단계 (릴리즈 계획)

### Phase 1 — 핵심 달력 + 타임라인 (v6.0) `P0`
| 항목 | 세부 내용 |
|------|-----------|
| 달력 월간 뷰 | 달력 그리드, 날짜 클릭 일정 등록, 프로젝트 바 표시 |
| 달력 주간 뷰 | 7일 확대 뷰 |
| 프로젝트 등록/편집/삭제 | CRUD + IndexedDB 저장 |
| 마일스톤 등록 | 프로젝트 하위 단계 관리 |
| 타임라인 기본 뷰 | 간트 차트, Today Line, 확대/축소 |
| 대시보드 요약 카드 | 상태별 프로젝트 수, 금주 마일스톤, 지연 경고 |
| 수주번호 연동 | order.js + 기존 수주맵 시스템 통합 |

### Phase 2 — 고급 인터랙션 (v6.1) `P1`
| 항목 | 세부 내용 |
|------|-----------|
| 드래그로 일정 변경 | 타임라인 바 + 달력 이벤트 드래그 |
| 진척률 자동 산출 | 아카이브 데이터 → 수주번호 매칭 → 투입시간 집계 |
| 담당자 필터 | 팀원/그룹별 필터링 |
| 팀원 리소스 맵 | 팀원별 할당 현황 히트맵 |
| 일정-수주번호 필터 | 프로젝트별 달력 필터링 |

### Phase 3 — AI + 연동 (v6.2) `P2`
| 항목 | 세부 내용 |
|------|-----------|
| AI 일정 인사이트 | 일정 충돌 감지, 과부하 경고, 최적 배분 제안 |
| 반복 일정 | 주간/격주/월간 반복 |
| 프로젝트 의존관계 | 선행/후행 화살표 연결 |
| Google Calendar 연동 | 외부 일정 불러오기 (선택적) |

## 8. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| **성능** | 50개 프로젝트, 200개 일정까지 렌더링 지연 없이 표시 |
| **저장** | IndexedDB 기반, 서버 불필요 (기존과 동일) |
| **호환** | Chrome 90+, Edge 90+, Firefox 90+ |
| **접근성** | 키보드 네비게이션 (달력 날짜 이동, 이벤트 선택) |
| **테마** | 기존 7종 테마 모두 지원 |
| **데이터 백업** | 프로젝트/일정 데이터 JSON 내보내기/가져오기 |
| **보안** | 로컬 전용, 외부 전송 없음 (AI 분석 시에만 API 호출) |

## 9. 성공 지표

| 지표 | 목표 |
|------|------|
| 프로젝트 등록 용이성 | 3클릭 이내 프로젝트 등록 완료 |

| 업무일지 연계 | 수주번호 기반 자동 진척률 오차 ±10% 이내 |
