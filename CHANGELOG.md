- **마일스톤 FK 제약 수정**: `createProjectFromOrder`에서 마일스톤이 프로젝트보다 먼저 생성되는 순서 버그 수정
- **체크리스트 스키마 불일치 해결**: 서버(phase별 items JSONB) ↔ 클라이언트(개별 row) 변환 레이어 추가
  - `chkGetByProject`: items JSONB → flat 개별 항목으로 변환
  - `toggleCheckItem`, `chkDel`, `createCheckItem`: 서버 모드 오버라이드
  - `GET /api/checklists/:id` 엔드포인트 추가
- **프로젝트 일괄 생성 API**: `POST /api/projects/full` — 프로젝트+마일스톤+체크리스트를 단일 트랜잭션으로 처리 (13회→1회 API 호출)

### 속도 최적화
- **API 병렬화**: dashboard(4곳), showProjectDetail, calendar의 순차 API 호출을 `Promise.all` 병렬로 변경

- **업무유형 분포**: 아카이브 데이터를 수주번호 기준으로 집계하여 프로젝트별 업무유형(A/B/D/G/M/S) 투입시간 스택 바 차트를 대시보드에 표시
- **아카이브 ↔ 마일스톤 연결**: 아카이브 레코드의 날짜를 마일스톤 기간과 매칭, 마일스톤별 실투입시간 자동 집계 → 프로젝트 상세 패널에 표시
- **진척률 히스토리**: IndexedDB v3 `progressHistory` 스토어 추가, 진척률 변경 시 일별 스냅샷 저장, 프로젝트 상세 패널에 Chart.js 번다운 차트 표시

### 지능형 경고
- **팀원 부하 경고**: 프로젝트 등록/편집 시 담당자 입력하면 기존 배정 프로젝트 수 실시간 표시 (초록/파랑/노랑/빨강 4단계)
- **이벤트-업무 충돌 감지**: 일정 등록 시 담당자의 동일 기간 프로젝트/일정 3건 이상 겹치면 경고 다이얼로그 표시

### 팀-프로젝트 연결
- **그룹 → 프로젝트 팀 구성**: 프로젝트 모달에 "그룹에서 가져오기" 드롭다운 추가, 그룹 선택 시 담당자 자동 입력. 반대로 담당자를 그룹으로 저장하는 버튼도 추가
- **마일스톤 자동 제안**: 아카이브에서 수주번호 기반으로 업무유형 전환 패턴을 감지하여 마일스톤 후보 자동 생성 (예: "D(개발) 집중 구간")

### 뷰 통합
- **트렌드에 프로젝트 컨텍스트**: 트렌드 분석 차트 아래에 해당 주차의 마일스톤 완료·프로젝트 시작/종료 이벤트 표시
- **달력에 아카이브 요약**: 월간 뷰에 아카이브 주간 데이터 배지 ("📊 320h/5명"), 주간 뷰에 요약 바 오버레이
- **AI 통합 인사이트**: 대시보드 AI 분석에 최근 4주 아카이브 데이터 포함 — 아카이브 vs 프로젝트 예실 비교, 미배정 인원 감지, 미착수 프로젝트 경고

### 구조 변경
- IndexedDB v2→v3: `progressHistory` 스토어 추가 (projectId, date 인덱스)
- `readAllArchiveRecords()`, `calcTaskDistByOrder()`, `calcHoursByMilestone()`, `suggestMilestones()`, `getWeeklyArchiveSummary()`, `getRecentArchiveWeeks()` 유틸 함수 추가
